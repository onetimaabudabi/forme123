# UI_GUIDELINES.md

# Forme UI Guidelines

## Philosophy

The interface should feel like a native Apple application.

Users should immediately perceive Forme as premium software.

Every screen should prioritize clarity over decoration.

---

# Design Language

Inspired by:

Apple Human Interface Guidelines

iOS

Fitness+

Health

Strava

Notion

Avoid copying any application directly.

---

# Visual Style

Minimal

Clean

Modern

Rounded

Premium

Comfortable spacing

No visual clutter.

---

# Colors

Primary

#3A86FF

Background

White

Dark Mode

Near Black

Text

Black

Secondary Text

Gray

Success

Green

Warning

Orange

Error

Red

Avoid excessive gradients.

Avoid neon colors.

---

# Typography

Use system fonts.

Hierarchy:

Large Title

Title

Headline

Body

Caption

Text should always remain readable.

---

# Corners

Cards

20-24px

Buttons

16-20px

Images

16-24px

Maintain consistency.

---

# Shadows

Soft shadows only.

Avoid strong shadows.

Prefer elevation through spacing.

---

# Buttons

Buttons should be:

Large

Rounded

Easy to tap

Consistent

Primary buttons use blue.

Danger buttons use red.

Secondary buttons remain neutral.

---

# Cards

Every card should have:

Rounded corners

Padding

Spacing

Clear hierarchy

Never overcrowd cards.

---

# Feed

Feed cards contain:

Avatar

Username

Timestamp

Caption

Media

Actions

Statistics

Comments Preview

Spacing should feel similar to Instagram but cleaner.

---

# Profile

Header contains:

Avatar

Username

Followers

Following

Friends

Streak

Join Date

Settings

Leaderboard

Posts appear below.

---

# Navigation

Bottom Navigation

Five Tabs

Home

Workout

Feed

Progress

Profile

Never exceed five main tabs.

---

# Animations

Animations should feel subtle.

Examples

Fade

Slide

Scale

Spring

Avoid excessive motion.

---

# Loading

Never show plain text:

Loading...

Instead use:

Skeletons

Activity Indicators

Logo Animation

Brand Splash Screen

---

# Empty States

Every empty screen should contain:

Illustration

Title

Description

Action Button

Never leave empty white screens.

---

# Images

Use rounded corners.

Support pinch-to-zoom (future).

Lazy load large images.

---

# Videos

Autoplay only when visible.

Mute by default.

Pause when leaving screen.

---

# Mobile Rules

Minimum touch area:

44x44 px

No overlapping buttons.

No hidden controls.

Support small screens.

---

# Dark Mode

Every screen must support:

Light

Dark

Avoid hardcoded colors.

---

# Accessibility

Readable fonts.

High contrast.

Screen reader labels.

Proper focus order.

Large tap targets.

---

# Consistency Rules

Every screen should reuse existing components.

Never invent new button styles.

Never introduce inconsistent spacing.

Maintain one visual language across the entire application.

---

# Future UI

Glassmorphism where appropriate.

Interactive avatars.

Animated statistics.

Dynamic themes.

Seasonal effects.

Apple Vision Pro support (future).

---

# Final Goal

Every screen should make users think:

"This feels like a premium app built by Apple."

That level of polish should be the target for every feature added to Forme.
